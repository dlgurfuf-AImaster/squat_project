# 하드웨어 회로 및 블루투스 패킷 통신 설계 문서

본 문서는 두 개의 MPU6050 센서와 블루투스 모듈을 아두이노 나노(Arduino Nano)에 연결하는 회로 설계 및 데이터 송신 프로토콜을 정의합니다.

---

## 1. 핀 맵 (Pin Mapping Table)

모든 모듈은 아두이노 나노의 5V와 GND를 공유하며, I2C 통신선(SDA, SCL)을 병렬로 연결합니다.

| 모듈 (Component) | 모듈 핀 (Pin) | 아두이노 나노 핀 (Arduino) | 비고 |

| :--- | :--- | :--- | :--- |

| **공통 전원** | VCC / GND | 5V / GND | 모든 모듈 공통 |

| **허리 센서 (mpu1)** | SDA / SCL | **A4 (SDA) / A5 (SCL)** | I2C 통신 공유 |

| | **ADO** | **GND** | **I2C 주소: '0x68'** |

| **허벅지 센서 (mpu2)** | SDA / SCL | **A4 (SDA) / A5 (SCL)** | I2C 통신 공유 |

| | **ADO** | **5V (또는 3.3V)** | **I2C 주소: '0x69'** |

| **블루투스 (HC-06 등)** | **TX** | **D2 (RX)** | SoftwareSerial 설정 |

| | **RX** | **D3 (TX)** | 아두이노 TX(3번) -> 모듈 RX |

---

## 2. 하드웨어 정교화 포인트 (Hardware Architecture)



1. **I2C 주소 분리 (ADO 핀 제어)**
- MPU6050은 내부적으로 ADO 핀의 전압 상태에 따라 주소가 결정됩니다.
- 허리 센서는 ADO를 GND에 묶어 '0x68'로 설정하고, 허벅지 센서는 ADO를 5V에 연결하여 '0x69'로 변환함으로써 단 하나의 I2C 버스에서 두 개의 센서를 독립적으로 제어합니다.

2. **소프트웨어 시리얼 분리**
- 아두이노의 하드웨어 시리얼(D0, D1)은 PC 업로드 및 디버깅용으로 비워두고, 블루투스 통신은 'SoftwareSerial(2, 3)'을 사용하여 데이터 오염 및 업로드 충돌을 방지했습니다.

---

## 3. 데이터 송신 프로토콜 (Serial Packet Protocol)

블루투스를 통해 수신단(앱/웹)으로 전송되는 데이터는 파싱(Parsing)이 용이하도록 실무형 패킷 구조를 채택했습니다.
- **샘플링 레이트**: 50Hz (`delay(20)`) - 실시간 운동 분석 및 각도 계산에 최적화된 주기

### 패킷 구조 예시

'''text
$1.23,0.45,9.81|-0.12,4.56,8.90

| 구분자 | 의미 | 설명 |

| $ | 패킷 시작 (Start of Packet) | 새로운 데이터 프레임의 시작을 알림 |

| , | 축 데이터 구분자 | X , Y , Z 가속도 데이터를 구별 |

| | | 센서 구분자 (Delimiter) | 앞 부분은 허리(mpu1), 뒷 부분은 허벅지(mpu2) 데이터 |

| \\n | 패킷 끝 (End of Packet) | 줄바꿈 문자로 하나의 데이터 전송 완료를 식별 |
